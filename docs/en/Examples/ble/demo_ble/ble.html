Ble Introduction

Overview

This example mainly intoduces how to use ble.

Steps for Usage

  • For BL602, enter customer_app/bl602_demo_event project, compile with script genromap , and dowload the bin file. Use stack_ble cli command to enable ble. This Demo doesn't start any ble behavior by default. If you need to start ble advertising or ble scan or initiate a ble connection, please use the cli commands to initiate the corresponding behavior.

  • For BL702, enter customer_app/bl702_demo_event project, compile with script genblem1s1 , and dowload the bin file. Ble has been enbled in this demo. This Demo doesn't start any ble behavior by default. If you need to start ble advertising or ble scan or initiate a ble connection, please use the cli commands to initiate the corresponding behavior.

  • For BL702L，enter customer_app/bl702l_demo_ble_peripheral project, compile with script genblem1s1p , and dowload the bin file. in this demo, ble has been enbled, adv is started, and low power mode is enabled. You can wake up BL702L by entering any cli command with a length of more than 10 bytes. The cli command only wakes up BL702L and disable low power mode, it will not be executed. BL702L will no longer actively enter into low power mode. If you do not need to enable adv at this time, you can disable adv by inputting cli command ble_stop_adv. If you want to enable BL702L low power mode, please input cli command pds_start.

The following commands are shared by BL602, BL702, and BL702L.

  • Use ble_init 、 ble_auth to register releated callbacks。

  • Use ble_start_adv 0 0 0x80 0x80 to start ble adv。

  • Use mobile app nRF Connect to scan nearby ble devices，and find out our devices.

  • Use ble_conn_update 0x6 0x6 0x0 0x1f4 to update connection parameters.

  • During SMP process, different commands are used according to different security level. Describe the cases of security level 2 and 3 as follows.

  • Use ble_security 2 to initiate SMP after ble connection is established.

    • Once Confirm passkey for xx:xx:xx:xx:xx:xx (public)：xxxxxx is printed out on serial port, input ble_auth_passkey_confirm to confirm passkey.

    • If Bonded with xx:xx:xx:xx:xx:xx (public) is printed out , SMP is completed successfully.

  • Use ble_security 3 to initiate SMP after ble connection is established.

    • Once Confirm passkey for xx:xx:xx:xx:xx:xx (public)：xxxxxx is printed out on serial port, input ble_auth_passkey_confirm to confirm passkey.

    • If Bonded with xx:xx:xx:xx:xx:xx (public) is printed out , SMP is completed successfully.

Steps for OTA

  • Step 1：Install APP

    • Download Bouffalo Lab BLE general apk (only support Android os currently) and install.

  • Step 2：Download the firmware with OAD service enabled

    • Take the project bl702_demo_event as an example，enable BLE OTA function by below steps：1)Add CONFIG_BT_OAD_SERVER=1 in the compilation script such as genblem1s1，then the content of the script is make CONFIG_BT=1 CONFIG_BLECONTROLLER_LIB=m1s1 CONFIG_BT_OAD_SERVER=1 -j; 2)Application layer calls API oad_service_enable to register OAD service. Execute the compilation script to generate bin file with OAD service and download it to BL702 board.

    • Input below cli commands via serial port to start ble adv.

      ble_init

      ble_start_adv 0 0 0xa0 0xa0

  • Step 3：Generate OTA file

    • Take the project bl702_demo_event as an example, add print information in version_cmd function in cli.c file to see whether OTA is successful. The content of compilation script is make CONFIG_BT=1 CONFIG_BLECONTROLLER_LIB=m1s1 CONFIG_BT_OAD_SERVER=1 -j , and generate bin file.

    • Open Dev Cube tool to download firmware. Choose related parameters as follows, and Firmware Bin is the OTA file generated in step 3. For Partition Table, select partition_cfg_1M.toml in which 1M flash is used. For Factory Params, select bl_factory_params_IoTKitA_32M.dts. Then, click Creat & Download button (The purpose of this step is to create an OTA image, no actual download, no need to connect the hardware board, and the download failure prompt can be ignored.). If there are prompts FW OTA bin is Done and FW OTA xz is Done in the log, then the OTA file is generated successfully.

    • Find out the generated OTA file in file directory of Dev cube. FW_OTA.bin.hash is a file without compression. FW_OTA.bin.xz.hash is a compressed file. In this example, FW_OTA.bin.xz.hash is used for OTA upgrade, and need to be put on the phone.

  • Step 4：OTA upgrade

    • On the phone, open APP to select related service, and scan ble devices (automatic scan is enabled by default).

    • Select device with name BL702-BLE-DEV to connect.

    • Click 设备升级 buttion in APP；

    • Select the file to be upgraded in the file directory, file is FW_OTA.bin.xz.hash.

    • Click SEND VER REQ button to send a version request to obtain information about current bin file.

    • Click START OTA to do OTA.

    • Reset BL702 device, then enter the cli command sysver, and find the added print information in step3 from the serial log. If the added info is found out, it means OTA is successful.

Cli command introduction

ble_init

  • Function：Before all ble cli operations, you need to enter this command first to register callbacks from application layer.

  • Parameters：None.

  • Example：Input ble_init.

ble_auth

  • Function：Register SMP callbacks from application layer.

  • Parameters：None.

  • Example：Input ble_auth.

ble_unpair

  • Function：Clear pairing information.

  • Parameter0: Address type of remote device.

  • 0：Public Address.

  • 1：Ramdom Address.

  • Parameter1: Address value of remote device. If set to 0, it means to clear all remote devices.

  • Example: Input ble_unpair 0 0.

ble_start_adv

  • Function: Start advertising

  • Parameter0: Advertising type

  • 0：connectable and scannable.

  • 1：non-connectable but scannable.

  • 2：non-connectable and non-scannable.

  • Parameter1: Advertising mode

  • 0：General discoverable.

  • 1：non discoverable.

  • 2：limit discoverable.

  • Parameter12: Minimum advertising interval. Range: 0x0020 to 0x4000. Unit:0.625 msec.

  • Parameter13: Maxmum advertising interval. Range: 0x0020 to 0x4000. Unit:0.625 msec.

  • Example：Input ble_start_adv 0 0 0x80 0x80.

ble_stop_adv

  • Function：Stop advertising.

  • Parameters: None

  • Example：Input ble_stop_adv

ble_start_scan

  • Function：Start ble scan to scan ble advertising devices.

  • Parameter0: Scan type

  • 0：Passive scan. No SCAN_REQ packets shall be sent.

  • 1：Active scan. SCAN_REQ packets may be sent.

  • Parameter1: Scanning filter policy.

  • bit0：1:Duplicate filtering enabled. 0:Duplicate filtering disabled.

  • bit1：1:Filter using whitelist. 0:Not filter using whitelist.

  • Parameter2: LE scan interval. Range: 0x0004 to 0x4000, Unit: 0.625 msec.

  • Parameter3: LE scan window. Range: 0x0004 to 0x4000, Unit: 0.625 msed. Scan window shall be less tha or equal to scan interval.

  • Example：Input ble_start_scan 0 0 0x80 0x40.

ble_stop_scan

  • Function：Stop ble scan.

  • Parameters: None

  • Example: Input ble_stop_scan.

ble_conn_update

  • Function: Updating connection parameters.

  • Parameter0: The minimum value of the connection interval, calculated as N * 1.25 ms. The range is from 7.5 ms to 4 s.

  • Parameter1: The maximum value of the connection interval, calculated as N * 1.25 ms. The range is from 7.5 ms to 4 s.

  • Parameter2: Slave latency for the connection in number of connection events. Range: 0x0000 to 0x01F3.

  • Parameter3: Supervision timeout for the LE link. Range: 0x000A to 0x0C80, Unit: 10 msec.

  • Example: After a successful connection, input ble_conn_update 0x28 0x28 0x0 0xf4.

ble_security

  • Function：Set security level for a connection.

  • Parameter0: Security level required.

  • 0：Only for BR/EDR special cases, like SDP.

  • 1：No encryption and no authentication.

  • 2：Encryption and no authentication (no MITM).

  • 3：Encryption and authentication (MITM).

  • 4： Authenticated Secure Connections and 128-bit key.

  • Example: After a successful connection, input ble_security 2.

ble_get_device_name

  • Function：Get local Bluetooth device name.

  • Parameters: None

  • Example: Input ble_get_device_name

ble_set_device_name

  • Function：Set local Bluetooth device name.

  • Parameter0：device name.

  • Example: Input ble_set_device_name bl602

ble_read_local_address

  • Function：Get local Bluetooth device address.

  • Parameters: None

  • Example: Input ble_read_local_address

ble_set_adv_channel

  • Function：Set ble advertising channel map.

  • Prameter0: Advertising channel map.

  • bit0: 1 means channel 37 shall be used, 0 means channel 37 shall not be used.

  • bit1: 1 means channel 38 shall be used, 0 means channel 38 shall not be used.

  • bit2: 1 means channel 39 shall be used, 0 means channel 39 shall not be used.

  • Example: Input ble_set_adv_channel 7

ble_connect

  • Function：Initiate an LE connection to a remote device.

  • Prameter0: The address type of remote device

  • 0：Public Device Address.

  • 1：Random Device Address.

  • 2：Public Identity Address.

  • 3：Random (static) Identity Address.

  • Prameter1: The address value of remote device, with more significant bytes first.

  • Example：Input ble_connect 0 18B905DE96E0

ble_disconnect

  • Function：Terminate an existing connection with the specified device.

  • Prameter0: The address type of remote device

  • 0：Public Device Address.

  • 1：Random Device Address.

  • Prameter1: The address value of remote device, with more significant bytes first.

  • Example：After a successful connection, input ble_disconnect 0 18B905DE96E0

ble_select_conn

  • Function: Among multiple connections, set a connection object as the current connection object.

  • Prameter0: The address type of remote device.

  • 0：Public Device Address.

  • 1：Random Device Address.

  • Prameter1: The address value of remote device, with more significant bytes first.

  • Example：After multiple devices are successfully connected, input ble_select_conn 1 5F10546C8D83 to set the selected connection object as the current connection object. Subsequent connection related operations such as ble_read will be performed on this connection.

ble_auth_cancel

  • Function: Cancle ongoing authenticated pairing.

  • Parameters: None

  • Example：During SMP process，input ble_auth_cancel

ble_auth_passkey_confirm

  • Function：Confirm the passkey received from the remote device. The passkey is also displayed on the remote device during the pairing process.

  • Parameters: None

  • Example：Once Confirm passkey for 48:95:E6:73:1C:1A (random): 745491` is printed out locally, input ``ble_auth_passkey_confirm to confirm passkey if the passkey is correct.

ble_auth_pairing_confirm

  • Function：Confirm an incoming pairing reqeust if user decides to accept the pairing. For example,

  • Parameters: None

  • Example：Once Confirm pairing for 00:1B:DC:F2:20:E9 (public) is printed out locally, input ble_auth_pairing_confirm to accept the pairing. .

ble_auth_passkey

  • Function：Reply with entered passkey during SMP process.

  • Parameter0：Entered passkey, Range:0 to 999999.

  • Example：Once Enter passkey for XX:XX:XX:XX:XX:XX (public) is printed out locally, input ble_auth_passkey 111111 to enter passkey.

ble_exchange_mtu

  • Function：Used by the client to set the ATT_MTU to the maximum possible value that can be supported by both devices when the client supports a value greater than the default ATT_MTU.

  • Parameters: None

  • Example: After a successful BLE connection, input ble_exchange_mtu

ble_discover

  • Function：Used by a gatt client to discover attributes on a server.

  • Parameter0: Disvoery type

  • 0：Discover primary services

  • 1：Discover secondary services

  • 2：Discover included services

  • 3：Discover characteristic

  • 4：Discover Descriptor

  • Parameter1: The UUID discovered. UUID is only used when discovery type is 0 or 1. When UUID is 0, discover all primary services or secondary services. When UUID is not 0, discover the service specified by the UUID.

  • Parameter2: Service start handle

  • Parameter3: Service end handle

  • Example: After a successful BLE connection, input ble_discover 0 0x1800 0x1 0xffff

ble_read

  • Function：Read attribute value by handle.

  • Parameter0: Attribute handle.

  • Parameter1: Attribute data offset.

  • Example: After a successful BLE connection, input ble_read 0x5 0

ble_write

  • Function：Write attribute value by handle.

  • Parameter0: Attribute handle.

  • Parameter1: Attribute data offset.

  • Parameter2: Length of the data.

  • Parameter3: Data to be written.

  • Example: After a successful BLE connection, input ble_write 0xf 0 2 0102

ble_write_without_rsp

  • Function：Write attribute value by handle without response.

  • Parameter0: Whether to sign data.

  • 0：not sign data.

  • 1：sign data.

  • Parameter1: Attribute handle.

  • Parameter2: Length of the data.

  • Parameter3: Data to be written.

  • Example: After a successful BLE connection, input ble_write_without_rsp 0 0xf 2 0102.

ble_subscribe

  • Function：Subscribe attribute value notification or indication.

  • Parameter0: Subscribe CCC handle.

  • Parameter1: Subscribe value handle.

  • Parameter2: Subscribe value.

  • 1：notification

  • 2：indication

  • Example: After a successful BLE connection, input ble_subscribe 0xf 0xd 0x1 if user wants to enable the characteristic's notification function.

ble_unsubscribe

  • Function：Unsubscribe attribute value notification or indication.

  • Parameters: None

  • Example: Input ble_unsubscribe

ble_set_data_len

  • Function：Suggest the maximum transmission payload size and maximum packet transmission time to be used for LL Data PDUs on a given connection.

  • Parameter0: Preferred maximum number of payload octets that local controller should include in a single link layer data channel PDU. Range: 0x001B - 0x00FB

  • Parameter1: Preferred maximum number of microseconds that the local controller should use to transmit a single link layer data channel PDU. Range: 0x0148 - 0x4290

  • Example：After a successful BLE connection, input ble_set_data_len 0xfb 0x0848.

ble_conn_info

  • Function：Get connected devices' information.

  • Parameters: None

  • Example：After a successful BLE connection, input ble_conn_info to get the remote device's information.

ble_disable

  • Function：Disable Bluetooth. This function is only available if there is no bluetooth activity at the moment. For example, if ble advertising or scan or connection exists, bluetooth cannot be disabled.

  • Parameters: None

  • Example：Input ble_disable

ble_set_tx_pwr

  • Function：Set ble tx power.

  • Parameter0: ble tx power value.

  • Example：Input ble_set_tx_pwr 0xa
