BL702L_Ble_Remote_Control_Demo

Overview

This Demo provides a ble remote control example, in which HOGP service is used.BL702L / BL704L RC DVK board can be used to run this Demo. Below is the work flow of this demo.

    1. Ble initialization.

    1. Register Battery service, HOGP service and Google voice service. GAP and GATT service are registered by default during initialization procedure in ble stack.

    1. Ble connectable advertising is started automatically, and low power mode is enabled.

    1. Then BL702L can be connected by Chromecast.

    1. After BL702L is connected by Chromecast, you can use BL702L to control Chromecast.

    1. In this demo, battery level is reported to Chromecast every 60s.

    1. After BL702L is connected by Chromecast, BL702L initiates a connection parameter update procedure to update the connection interval to 40ms. Once voice key is pressed, BL702L will initiate a connection parameter update procedure to update the connection interval to 20ms to transmit voice data to Chromecast.

About Test

  • Pins Configuration

    In ble_rc_kys_init function, we use a 4*4 keyscan. Below pins are used in key scan. uint8_t row_pins [] = {10, 25, 24, 23};

    In this row_pins and col_pins configuration, the keys in yellow blocks are not used (avoid confliction with JTAG pin during development stage).

    Please press AC HOME key and AC BACK key at the same time to make BL702L do unpair and start advertising.

  • Voice Test

    To capture voice with PDM interface, please connect IO8 to PCLK and connect IO3 to PIN.

    To capture voice with AMIC interface, please connect IO8 to AIN2 and connect IO3 to AIN3.
