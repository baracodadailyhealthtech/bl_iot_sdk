Mesh

Overview

This example mainly introduces how to use ble mesh.

Steps to use Node

  • Download Bouffalo Lab BLE mesh apk (Currently only supports Android system) and install.

  • Enter customer_app/bl602_demo_event project or customer_app/bl702_demo_event project or customer_app/bl702l_demo_mesh project . Compile with genblemeshmodel, and download the BIN file.

  • Run Mesh related cli commands on the serial port:

    • BL602：

      #stack_ble

      #blemesh_init

      #blemesh_pb 2 1

    • BL702 or BL702L：

      #blemesh_init

      #blemesh_pb 2 1

  • Open the App, select the Network column and click Add.

  • In the scanning interface, select the device to be added.

  • Click the IDENTIFY option.

  • Click the PROVISION option.

  • Select No OOB in the pop-up diaglog, and click ok.

  • Click ok in the pop-up diaglog.

  • The interface is automatically transferred to the Network column, click on the device that has been successfully connected.

  • Select the Elements option, and click its drop-down button.

  • Select Generic on off Server option.

  • Select BIND KEY option

  • Click Application key 1

  • Click ONOFF option to control the switch of the LED.

  • The following information can be seen in the serial port, indicating that the control of the LED is successful.

  • Click SUBSCRIBE option.

  • Click Create a new group to subscribe option.

  • Follow the same method to add another device node. After the addition is successful, in the Groups option, click the group that has been successfully created.

  • In the group, select ON/OFF, and the log is printed on the serial port to prompt the LED switch information, indicating that the mesh networking is successful.

Steps to use Provisioner

The following configuration options need to be enabled when compiling.

CONFIG_BT_MESH_PROVISIONER=1

  • Start the node that has not joined the network and enter the state of waiting to join the network.

    1. Initialize BLE.

      stack_ble

    2. Initialize MESH.

      blemesh_init

    3. Send PB-ADV Unproved beacon.

      blemesh_pb 1 1

  • Provisioner

    1.Start provisioner and add nodes which are not in the mesh network.

    1.1 Initialize BLE.

    stack_ble

    1.2 Initialize MESH.

    blemesh_pvnr_init

    Follows are the printed log:

    # blemesh_pvnr_init

    Local node provisioned, net_idx 0x0000 address 0x0001

    Network key:8fe7deea92943001567251dc1f9b0f03

    Dev key:89c2f18f9f286661c6bbb1430d7aadb7

    Provisioning completed

    1.3 Discover unprovisioned devices, and UUID information of the unprovisioned devices will be printed out later on.

    blemesh_beacon_listen 0x01

    1.4 Add specified equipment to the network.

    blemesh_provision_adv 07af000011112222333318b905de96c2 0 3 0

    P1：“07af000011112222333318b905de96c2" is device UUID.

    P2：0：net_idx

    P3：3：Specifies the unicast address of the device to be provisioned.

    P4：0：attention_duration

    The following prompt information will be printed after the device is successfully provisioned.

    Node added:

    net_idx:[0]

    uuid:[07af000011112222333318b905de96c2]

    addr:[3]

    num_elem:[1]

    Device is successfully provisioned.

    1. Set up node model server

    2.1 Get composition data

    blemesh_get_comp 0x0000 0x0003 0x00

    P1：net index.

    P2：dst address.

    P3：page number.

    2.2 Add appkey

    blemesh_app_key_add 0x0000 0x0003 0x0000 0x0000

    P1：net index.

    P2：dst address.

    P3：Network key index.

    P4：Application key index.

    2.3 Bind Generic onoff server model

    blemesh_mod_app_bind 0x0000 0x0003 0x0003 0x0000 0x1000

    P1：net index.

    P2：dst address.

    P3：element address.

    P4：Application key index.

    P5：Model ID.

    1. Provisioner use client to control node's server

    3.1 Register callback

    blemesh_gen_oo_cli reg

    3.2 Get Generic onoff server status

    blemesh_gen_oo_cli get 0x0000 0x8201 0x00 0x0003 0x0000

    3.3、Set Generic onoff server

    Set onoff status to 1

    blemesh_gen_oo_cli set 0x0000 0x8202 0x00 0x0003 0x0000 0x00 0x01 0x00 0x00 0x00

    Set onoff status to 0

    blemesh_gen_oo_cli set 0x0000 0x8202 0x00 0x0003 0x0000 0x00 0x00 0x01 0x00 0x00

    1. Remove node

    blemesh_node_reset 0x0000 0x0003

    1. Group subscription operation

    Add：

    blemesh_mod_sub_add 0x0000 0x0003 0x0003 0xc000 0x1000

    P1：net index.

    P2：dst address.

    P3：element address.

    P4：subscription address.

    P5：Model ID

    Delete：

    blemesh_mod_sub_del 0x0000 0x0003 0x0003 0xc000 0x1000

    P1：net index.

    P2：dst address.

    P3：element address.

    P4：subscription address.

    P5：Model ID

    1. Vendor model operation

    Need to set CONFIG_BT_MESH_MODEL_VENDOR_CLI to 1 to enable the function of vendor cli model.

    6.1 Bind Vendor server model on node

    blemesh_mod_app_bind 0x0000 0x0003 0x0003 0x0000 0x0002 0x07AF

    P1：net index.

    P2：dst address.

    P3：element address.

    P4：Application key index.

    P5：Model ID. (0x0002) bouffalolab vendor model id.

    P6: Company ID.(0x07AF) bouffalolab company id.

    6.2 Provisioner send data to Node

    blemesh_vendor_cli 0x0000 0x0008 0x00 0x0003 0x0000 001234

    P1：Application key index

    P2：opcode

    P3：msg_role (currently unused)

    P4：addr

    P5：net index

    P6：status
