BL IOT SDK Development Guide

Overview

The BL702L/704L series chip is a general purpose microcontroller which is developed by Bouffalo Lab Technology, and based on RISC-V processor. It has a main frequency of up to 128MHz and rich peripherals. It also supports BLE 5.0, Zigbee 3.0 and other functions, and can be widely used in IoT and other low-power fields.

The BL IOT SDK is an IOT software development kit specially provided by Bouffalo Lab team for BL60X, BL70X and subsequent chips.

This article uses a simple example to show how to build a development environment, compile example code, burn firmware and view output in a Windows environment, aiming to guide users to build a complete BL70X IOT application development environment.

Preparation

Before starting this example, the required development environment includes:

  • Hardware

    • BL702L/704L DVK board

    • USB cable

    • PC (Windows OS)

  • Software

    • MSYS2 (Windows compilation tool)

    • Freedom Studio (Windows debug tool)

    • BLDevCube (Firmware programming tool)

Hardware Connection

The BL702L/704L DVK board reserves rich communication interfaces for the chip, and supports programming firmware and debugging software through UART, JTAG and other connection methods.

The BL702L/704L DVK board integrates a BL702 chip, which is used to realize the conversion between USB-CDC (UART) and USB-CKLink (JTAG), and can easily realize software programming, serial communication and online debugging and other functions.

Software Installation

Before starting to compile the project, please complete the installation of all the following tools correctly. The specific steps and environment variable settings are as follows:

  • MSYS2 MSYS32 Windows Platform Installation and Compilation

  • BLDevCube BLDevCube Firmare Programming

重要

Please pay attention to the settings of environment variables, drivers and other information in the document, which will avoid abnormal errors in subsequent operations.

Project Compilation

This section describes how to compile the bl702l_demo_event project under the customer_app directory.

  1. Open the MSYS2 platform and enter the local bl_iot_sdk/customer_app/bl702l_demo_ble_peripheral folder

  • Command: cd /c/bl_iot_sdk/customer_app/bl702l_demo_ble_peripheral/

  1. Execute the script genblem1s1p to start compiling the project

  • Command: ./genblem1s1p

  1. After the compilation is complete, a build_out folder with the following files are added to the current directory:

  • **bl702l_demo_ble_peripheral.bin: ** Binary firmware for programming into the chip

  • **bl702l_demo_ble_peripheral.elf: ** Executable file for debugging software

  • **bl702l_demo_ble_peripheral.map: ** Map file for analysis of functions and memory usage

Bouffalo Lab Parse Tool For Windows is a map file analysis software provided by Bouffalo Lab team, which can be used for memory analysis

Firmware Programming

This section introduces how to use BLDevCube integrated development tool to program firmware into BL702L/704L DVK board.

1. Let the chip enter programming mode

1.1 After connecting to BL702L/704L DVK board with USB cable

  • First press the Boot key on the board, do not release

  • Then press the RST key on the board and release it; at this time the chip enters UART programming mode

  • Finally release the Boot key

2. Download firmware using BLDevCube

2.1 Open BLDevCube.exe, select BL702L/704L in the drop-down box of Chip, click Finish and enter the main interface of Dev Cube

2.2 After entering the main interface, enter the IOT download interface by default

2.3 Configure BLDevCube

  • Configuration information includes:

    • Interface: It is used to select the communication interface for downloading, here select Uart.

    • Port/SN: When Uart is selected for downloading, here select the COM port connected to the chip, you can click the Refresh button below to refresh the COM port.

    • Uart Rate: When Uart is selected for downloading, fill in the baud rate, the default value is 2000000 bps.

    • Chip Erase: Choose whether to erase the entire Flash before downloading, the default setting is False.

    • Xtal: Select the crystal oscillator frequency when downloading, if the circuit board does not have a soldered crystal oscillator, select the internal RC32M.

  • Firmware information includes:

    • partition table: Use the partition table in the partition folder of the corresponding chip type under the Dev Cube folder, default select ...\chips\bl702l\partition\partition_cfg_1M.toml

    • firmware: The path of bin file generated by user compilation ...\bl_iot_sdk\customer_app\bl702l_demo_ble_peripheral\build_out\bl702l_demo_ble_peripheral.bin

    • dts: Use the device tree in the device_tree folder of the corresponding chip type under the Dev Cube folder, default select ...\chips\bl702l\device_tree\bl_factory_params_IoTKitA_32M.dts

2.4 After completing the above configuraiton correctly, click Create & Download for downloading，after the successful download, you can see the status bar turns green

3. Run firmware

  • Open the serial port tool installed on the PC, select 2000000 baud rate

  • Press the RST key on the DVK board

  • The program is executed correctly, and the PC-side serial port tool will see the following print

注解

During the burning, if there are abnormalities such as unable to burn, burning failed, and no printing after burning successfully, refer to BL702 FAQ Summary .

Debug Software

This section describes how to use Freedom Studio to debug code.

1. Hardware connection

1.1 Use jumper caps to connect the following 4 groups of JTAG signals of the J3 connector on the BL702L/704L DVK board:

  • IO0(J3.10) - TMS(J3.12)

  • IO1(J3.09) - TDI(J3.11)

  • IO2(J3.13) - TCK(J3.15)

  • IO7(J3.18) - TDO(J3.16)

2. Install and configure CKLink

2.1 Download the latest version CKLink driver from T-HEAD official website

2.2 Unzip the downloaded CKLink installation package, double-click Setup, and click next all the way. After the installation is complete, there will be a T-HeadDebugServer icon on the desktop

2.3 After connecting the JTAG pins of the board with the CKLink debugger, click the triangle button, if it turns into a circle, it means that the JTAG connection is successful. If it fails, you need to check if the JTAG connection is correct

3. Install and use Freedom Studio

3.1 Download and unzip Freedom Studio, download website https://github.com/sifive/freedom-studio/releases, version FreedomStudio-2019-08-2

3.2 Run FreedomStudio.exe

3.3 Click File | Import menu and open Import dialog box, import project C/C++ Existing Code as Makefile Project

3.4 Click Browse button and select project folder, make sure the Project Name is bl_iot_sdk

3.5 Open the Debug Configurations dialog box as shown below

3.6 If you want to see the current code running status, please select bl_iot_sdk_debug_freedom_studio_win_attach_bl702l.launch; If you want to execute the code from scratch, please select bl_iot_sdk_debug_freedom_studio_win_bl702l.launch

重要

Before using bl_iot_sdk_debug_freedom_studio_win_bl702l.launch, the chip needs to enter the UART programming mode.
